안녕하세요 게임공학과 윤진우 입니다
제가 만들어 볼 앱은 게임 인포라고 하는 
게임 정보 검색 앱 입니다.

목차는 
개발, 사용목적
앱 UI 구조
뷰 설명
사용 API
개발 범위로 구성 되어 있습니다.

먼저 개발 및 사용 목적 입니다.
개발 목적은 사용자가 원하는 게임을 찾거나 혹은
여러 회사의 게임을 찾기 위해 앱을 개발하려고 합니다.

사용목적은 원하는 게임이나 게임의 정보 및 
회사이름을 통해 제대로 된 정보를 빠르게 찾는목적 입니다.

앱 구조 입니다.
크게 메뉴선택, 검색, 검색결과, 상세내용 4가지로 나뉘며
자세한 설명은 뷰 설명에서 하도록 하겠습니다.

먼저 메뉴선택입니다.
상단에는 게임과 관련된 이미지가 뜨고
아래에는 4개의 메뉴를 선택할 수 있습니다. 

검색에선 
순서대로 게임검색, 회사검색, 년도 검색, 상세 검색으로
나뉩니다.

게임검색은 상단에 게임관련된 이미지와 게임 검색창, 그리고 검색 버튼이 있습니다.
회사검색은 중앙에 유명 회사 피커뷰가 있으며 피커뷰에 따라 상단의 이미지가 바뀝니다.
피커뷰에 없는 회사를 찾을 경우 피커뷰를 없음에 두고 아래 회사를 검색 하면 됩니다.
년도검색에서는 역시 중앙에 년도 피커뷰가 존재, 위의 이미지는 년도별 goty를 수상한 게임의 이미지가
뜹니다. 아래에는 게임의 등급 피커 뷰가 뜹니다. 역시 이를 이용해 검색합니다.
상세검색에서는 게임 이름, 회사, 년도 추가적으로 이용가 등급까지 입력 후 검색 할 수 있습니다.

검색결과는 위에 직전에 검색된 내용을 가지고 테이블 형태로 나타나게 되며 그 위에는 직전에 입력된
검색 내용이 뜨게 됩니다.

검색된 테이블 뷰를 누르게 되면 해당 게임 이미지와 함께 아래에 게임의 상세정보가 나타납니다.

사용된 API는 게임물관리위원회 에서 제공하는 게임물 3.0이란 API 입니다.
또한 추가적으로 사용될 API를 찾고 있습니다.

개발범위는 최소, 목표, 최대로 나뉘었으며
최소인 경우 게임이름을 검색하여 게임의 정보를 나타내는걸 기본으로 두고 있습니다.
목표인 경우 제안서 대로 UI를 만들고 위에 올려둔 회사나 날짜 등의 다양한 정보로 접근, 결과를 도출
상세정보를 통한 정확한 검색이 되도록 만들려고 합니다.
마지막으로 최대범위에선 더 많은 다수의 API를 활용하여 장르나 캐릭터 등의 정보를 추가해 더 많은 
게임의 정보를 나타내는 것이며 그 밖에 게임 이미지나 관련 자료등의 연결등을 목표로 두고 있습니다.
